Git - Hệ thống quản lý phiên bản
- lưu trạng thái của file dưới dạng lịch sử cập nhật
- có thể đưa file đã chỉnh sửa về trạng thái cũ hay hiển thị sự khác biệt ở những dòng chỉnh sửa
- khi chia sẻ file trong một nhóm, server sẽ hiện ra cảnh báo khi có hiện tượng ghi đè/ xoá mất các nội dung 

- Repository là nơi ghi lại trạng thái thay đổi của file, được chia thành local repository và remote repository.
	+ Local repository: là repository trên máy tính cá nhân
	+ Remote repository: là repository trên server, nơi nhiều người có thể truy cập. Được sử dụng khi ta muốn upload các công việc của bản thân hoặc muốn truy cập công việc của người khác.

1/ git init: tạo một kho chứa git mới
2/ git status: để check trạng thái các file trong git
3/ git add ten_file: để cập nhật file vào staging area
4/ git status: để check lại xem các file mong muốn đã ready để được commit hay chưa

- Commit để ghi lại việc thêm/thay đổi file vào repository. Khi thực hiện commit, trong repository sẽ ghi lại sự khác biệt từ trạng thái đã commit lần trước đến trạng thái commit gần nhất. Các commit được ghi lại theo thứ tự thời gian.

5/ git commit -m "Mục đích của việc commit"
6/ Sau khi đã commit file vào local repository, dùng git log để kiểm tra lại xem file đã được commit chưa
7/ Để đồng bộ local repository lên server, dùng lệnh git remote add origin để xác định địa chỉ trên online repository mà git cần push file lên
8/ Dùng lệnh git push -u origin master để push file lên branch master trên github
9/ Lúc này source code đã có trên git hub
10/ git checkout commit: để trở về một version nào đã được commit trước đó
11/ git show: để xem đã có thay đổi gì 
12/ git diff: xem net difference giữa các version đã commit

13/ git branch new_branch: để tạo một nhánh khác 
14/ git checkout new_branch: để chuyển sang branch mình muốn dùng
15/ git merge: đồng bộ hoá các thay đổi của nhiều bên khác nhau
16/ Khi có conflict, phải resolve bằng tay, phải chọn phần nào mình muốn push lên 
17/ pull-request: tool để review code trước khi cho quyền merge
